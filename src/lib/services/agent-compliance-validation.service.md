# Agent Compliance Validation Service

## Purpose
The `Agent Compliance Validation Service` is designed to validate agents against predefined compliance policies. It assesses content according to various regulations, providing detailed violation reports and remediation suggestions.

## Usage
This service integrates with external compliance APIs and utilizes Supabase for data storage and Redis for caching. It helps ensure that the content generated by agents meets regulatory and platform policies.

## Parameters/Props

### PolicyRule Interface
- `id` (string): Unique identifier for the policy rule.
- `name` (string): Name of the policy rule.
- `category` ('content' | 'privacy' | 'safety' | 'regulatory' | 'platform'): The category to which the policy belongs.
- `severity` ('low' | 'medium' | 'high' | 'critical'): The severity level of the policy.
- `enabled` (boolean): Indicates if the rule is active.
- `patterns` (string[]): Regex patterns or keywords for rule validation.
- `description` (string): A brief description of the policy rule.
- `regulatoryBasis` (optional string): The basis or law that requires compliance.
- `lastUpdated` (Date): Timestamp for when the policy was last updated.
- `version` (string): Version of the policy rule.

### ComplianceViolation Interface
- `ruleId` (string): ID of the violated rule.
- `ruleName` (string): Name of the violated rule.
- `category` (string): Category of the violation.
- `severity` ('low' | 'medium' | 'high' | 'critical'): Severity of the violation.
- `description` (string): Description of the violation.
- `matchedContent` (string): Content that caused the violation.
- `confidence` (number): Confidence score of the violation detection.
- `location` (object):  
  - `startIndex` (number): Start index of the violation in the content.
  - `endIndex` (number): End index of the violation in the content.
  - `context` (string): Context surrounding the violation.
- `suggestedRemediation` (string[]): Suggestions for remediation.
- `regulatoryImplications` (optional string): Possible regulatory implications.

### ExternalComplianceResult Interface
- `provider` ('moderatrix' | 'aws-comprehend' | 'azure-content-safety'): Source of the compliance result.
- `verdict` ('compliant' | 'violation' | 'review_required'): Overall verdict of compliance.
- `confidence` (number): Confidence score of the verdict.
- `categories` (string[]): List of categories applicable to the verdict.
- `details` (object): Detailed results from the provider.
- `processingTime` (number): Time taken to fetch the compliance results.

### ComplianceReport Interface
- `id` (string): Unique report identifier.
- `agentOutputId` (string): Related agent output ID.
- `agentId` (string): ID of the agent validated.
- `userId` (string): ID of the user associated with the agent.
- `timestamp` (Date): Date and time of the report generation.
- `overallStatus` ('compliant' | 'violations_detected' | 'review_required' | 'blocked'): Summary status of compliance.
- `riskScore` (number): Calculated risk score based on violations.
- `violations` (ComplianceViolation[]): Array of detected violations.
- `externalResults` (ExternalComplianceResult[]): Array of results from external compliance checks.
- `metadata` (object):  
  - `contentType` (string): Type of the content validated.
  - `contentLength` (number): Length of the validated content.
  - `processingDuration` (number): Duration taken for processing.
  - `validationMethods` (string[]): Methods used for validation.
- `auditTrail` (object):  
  - `validatedBy` (string): Who validated the report.
  - `validatedAt` (Date): When the report was validated.
  - `reviewedBy` (optional string): Who reviewed the report (if applicable).
  - `reviewedAt` (optional Date): When the report was reviewed.
  - `status` (string): Status of the audit trail.
- `remediationRequired` (boolean): Indicates if remediation is needed.
- `blockingViolations` (boolean): Specifies if any violations are blocking compliance.
