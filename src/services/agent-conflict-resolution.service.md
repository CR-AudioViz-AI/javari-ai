# Agent Conflict Resolution Service Documentation

## Purpose
The Agent Conflict Resolution Service is designed to handle and resolve conflicts between agents in a multi-agent system. It provides mechanisms for identifying various types of conflicts, voting sessions for resolution, and applying strategies to resolve disputes effectively.

## Usage
To utilize the Agent Conflict Resolution Service, you will need to construct `AgentConflict` objects representing conflicts, initiate `VotingSession` for resolution, and implement resolution strategies to resolve conflicts based on agent votes.

## Interfaces and Enums

### AgentConflict
Represents a conflict between agents.

#### Properties
- **id**: `string` - Unique identifier for the conflict.
- **type**: `ConflictType` - Type of the conflict (e.g., output contradiction, resource contention).
- **agentIds**: `string[]` - List of agent IDs involved in the conflict.
- **timestamp**: `Date` - Time at which the conflict occurred.
- **description**: `string` - Brief description of the conflict.
- **severity**: `ConflictSeverity` - Severity level of the conflict.
- **metadata**: `Record<string, any>` - Additional metadata related to the conflict.
- **outputs**: `AgentOutput[]` (optional) - Outputs generated by agents involved in the conflict.
- **resourceContention**: `ResourceContention` (optional) - Information on resource contention if applicable.

### ConflictType
Enumerates the types of conflicts that can occur.
- **OUTPUT_CONTRADICTION**
- **RESOURCE_CONTENTION**
- **PRIORITY_COLLISION**
- **CONSENSUS_FAILURE**
- **PERFORMANCE_DEGRADATION**

### ConflictSeverity
Defines the levels of conflict severity.
- **LOW**
- **MEDIUM**
- **HIGH**
- **CRITICAL**

### ResourceContention
Details the contention over resources.
#### Properties
- **resource**: `string` - The resource that is contended.
- **requestingAgents**: `string[]` - List of agents requesting the resource.
- **availableCapacity**: `number` - Available capacity of the resource.
- **totalDemand**: `number` - Total demand for the resource.

### Vote
Represents a vote cast by an agent.
#### Properties
- **agentId**: `string` - ID of the agent casting the vote.
- **choice**: `string` - Agent's choice in the vote.
- **confidence**: `number` - Confidence level in the choice.
- **weight**: `number` - Weight of the vote.
- **reasoning**: `string` (optional) - Optional reasoning for the vote.
- **timestamp**: `Date` - Time the vote was cast.

### VotingSession
Configuration for a voting session to resolve conflicts.
#### Properties
- **id**: `string` - Unique ID of the voting session.
- **conflictId**: `string` - ID of the related conflict.
- **participants**: `string[]` - List of participant agents.
- **options**: `string[]` - Choices available during the voting.
- **strategy**: `VotingStrategy` - Strategy employed in the voting session.
- **weights**: `Map<string, number>` - Weights assigned to each participant.
- **timeout**: `number` - Voting session timeout in milliseconds.
- **startTime**: `Date` - Start time of the voting session.

### VotingStrategy
Enumerates available voting strategies.
- **MAJORITY**
- **WEIGHTED_MAJORITY**
- **EXPERTISE_WEIGHTED**
- **CONSENSUS**
- **HYBRID**

## Examples
1. Creating a conflict:
   ```typescript
   const conflict: AgentConflict = {
       id: 'conflict_001',
       type: ConflictType.OUTPUT_CONTRADICTION,
       agentIds: ['agent_1', 'agent_2'],
       timestamp: new Date(),
       description: 'Output of agents contradicting each other',
       severity: ConflictSeverity.HIGH,
       metadata: {},
   };
   ```

2. Setting up a voting session:
   ```typescript
   const votingSession: VotingSession = {
       id: 'vote_001',
       conflictId: conflict.id,
       participants: ['agent_1', 'agent_2'],
       options: ['solution_a', 'solution_b'],
       strategy: VotingStrategy.MAJORITY,
       weights: new Map([['agent_1', 1], ['agent_2', 2]]),
       timeout: 30000,
       startTime: new Date(),
   };
   ```

This documentation serves as a reference for developers implementing the Agent Conflict Resolution Service in their applications.
