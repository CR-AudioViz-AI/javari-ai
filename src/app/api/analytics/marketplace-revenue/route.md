# Marketplace Revenue Analytics API Documentation

## Purpose
The Marketplace Revenue Analytics API provides endpoints for retrieving detailed revenue analytics related to marketplace transactions. It allows authorized users to access revenue metrics, including total revenue, transaction counts, and performance breakdowns by agent and category.

## Usage
The API is designed to be used with Next.js's API routes, specifically leveraging Supabase for database interactions and cookie-based authentication for access control. The primary functionality is encapsulated in the `route.ts` file within the API directory of the application.

## Parameters/Props
### Request
- **Authorization**: The request must include a `supabase-auth-token` cookie for authentication purposes.

### Response
The API returns a `RevenueMetrics` object containing the following properties:
- `totalRevenue` (number): The total revenue generated from marketplace transactions.
- `totalCommission` (number): Total commission earned from all transactions.
- `totalTransactions` (number): The total number of transactions that occurred.
- `averageTransactionValue` (number): The average value per transaction.
- `growthRate` (number): The growth rate of revenue over a specified period.
- `topAgents` (Array): List of top-performing agents with their metrics:
  - `agentId` (string): Unique identifier for the agent.
  - `agentName` (string): Name of the agent.
  - `revenue` (number): Revenue generated by the agent.
  - `transactions` (number): Number of transactions handled by the agent.
  - `commission` (number): Total commission earned by the agent.
- `categoryBreakdown` (Array): Breakdown of revenue by category:
  - `category` (string): Name of the category.
  - `revenue` (number): Revenue from the specific category.
  - `transactions` (number): Number of transactions in the category.
  - `percentage` (number): Percentage of total revenue from this category.
- `timeSeriesData` (Array): Revenue data over time:
  - `period` (string): Time interval for the data point.
  - `revenue` (number): Revenue for the period.
  - `transactions` (number): Number of transactions during the period.
  - `commission` (number): Commission earned during the period.

## Return Values
The API returns a JSON object containing the `RevenueMetrics` data if the user is authenticated and authorized. If authentication fails, a null response will be issued.

## Examples

### Example Request
```http
GET /api/analytics/marketplace-revenue
Cookie: supabase-auth-token=<your-token>
```
