# Semantic Agent Search API Documentation

## Purpose
The Semantic Agent Search API is designed to perform efficient searches for AI agents based on user queries. It utilizes vector embeddings generated by OpenAI to optimize search results by relevance and similarity.

## Usage
This API endpoint can be called to search for agents in the marketplace based on the specified parameters. The endpoint expects a structured request and will return a list of agents that match the given criteria.

## Endpoint
```
POST /api/marketplace/agents/search
```

## Parameters/Props
The API accepts the following parameters in the request body:

- `query` (string, required): A user-defined search query with a minimum length of 1 and a maximum of 500 characters.
  
- `limit` (number, optional): Specifies the maximum number of results to return. Should be between 1 and 50. Defaults to 20.
  
- `threshold` (number, optional): A relevance threshold for filtering results. Should be between 0 and 1. Defaults to 0.7.
  
- `category` (string, optional): An optional category to narrow down the search results.
  
- `tags` (array of strings, optional): A list of tags to further refine the search. Each tag can be up to 500 characters.

## Return Values
The API returns a JSON object containing the following fields:

- `agents` (array): An array of agent objects that match the search criteria. Each agent object includes relevant metadata such as ID, name, description, and additional properties.

- `totalResults` (number): The total number of agents that matched the search query.

- `success` (boolean): Indicates whether the search operation was successful.

## Examples

### Example Request
```json
POST /api/marketplace/agents/search
{
  "query": "AI for automated reporting",
  "limit": 10,
  "threshold": 0.8,
  "category": "reporting",
  "tags": ["automation", "AI", "reporting"]
}
```

### Example Response
```json
{
  "success": true,
  "totalResults": 5,
  "agents": [
    {
      "id": "agent_12345",
      "name": "AutoReport AI",
      "description": "An AI agent that automates reporting tasks seamlessly."
    },
    {
      "id": "agent_67890",
      "name": "QuickReport AI",
      "description": "AI assistant designed for instant report generation."
    }
  ]
}
```

## Rate Limiting
This endpoint is rate-limited to prevent abuse, allowing a maximum of 500 unique tokens per minute.

## Error Handling
If an error occurs during processing, such as failure to connect to the OpenAI service or validation errors, appropriate error messages will be returned with corresponding status codes.
