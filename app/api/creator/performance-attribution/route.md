# Creator Performance Attribution API Documentation

## Purpose
The Creator Performance Attribution API is designed to assess the performance of creators by attributing revenue to various marketing touchpoints. It enables creators to gain insights into their revenue streams and optimize their marketing strategies based on data-driven attribution models.

## Usage
This API provides endpoints for querying performance attributions, tracking new touchpoints, and updating attribution models. It is built using TypeScript and integrates with Supabase for data storage.

### Endpoints
1. **Get Attribution**: Retrieve performance attribution for a specific creator.
2. **Track Touchpoint**: Log a new marketing touchpoint for a creator.
3. **Update Attribution Model**: Modify the attribution model and its parameters for a creator.

## Parameters/Props

### Attribution Query Parameters
- `creatorId` (string, required): UUID of the creator whose performance is being queried.
- `timeframe` (enum, optional): Time range for attribution calculation (default: '30d'). Options: ['7d', '30d', '90d', '1y'].
- `model` (enum, optional): Attribution model to use (default: 'linear'). Options: ['first-touch', 'last-touch', 'linear', 'time-decay', 'position-based'].
- `includeCollaborations` (boolean, optional): Include collaboration revenue (default: true).
- `includeReferrals` (boolean, optional): Include referral revenue (default: true).

### Touchpoint Tracking Parameters
- `creatorId` (string, required): UUID of the creator.
- `touchpointType` (enum, required): Type of touchpoint being tracked. Options: ['direct_sale', 'referral', 'collaboration', 'content_view', 'social_share'].
- `revenue` (number, required): Revenue generated by the touchpoint (must be 竕･ 0).
- `customerId` (string, optional): UUID of the customer associated with the touchpoint.
- `projectId` (string, optional): UUID of the project associated with the touchpoint.
- `metadata` (object, optional): Additional metadata associated with the touchpoint.
- `timestamp` (string, optional): Timestamp of the touchpoint in ISO 8601 format.

### Attribution Model Update Parameters
- `creatorId` (string, required): UUID of the creator.
- `model` (enum, required): New attribution model to apply.
- `weights` (object, optional): Custom weights for different touchpoint types:
  - `directSales` (number, optional): Weight for direct sales (0-1).
  - `referrals` (number, optional): Weight for referrals (0-1).
  - `collaborations` (number, optional): Weight for collaborations (0-1).
  - `contentEngagement` (number, optional): Weight for content engagement (0-1).
- `decayRate` (number, optional): Decay rate for time-based models (0-1).
- `lookbackWindow` (number, optional): Days to look back for considering touchpoints (1-365).

## Return Values
- **Attribution Result**: Contains total and attributed revenue, a breakdown of touchpoints, and a list of top-performing touchpoints.
- **Touchpoint Data**: Includes identifiers, revenue, and metadata for each touchpoint logged.

## Examples

### Get Attribution Example
```typescript
const attribution = await api.getAttribution({
  creatorId: '123e4567-e89b-12d3-a456-426614174000',
  timeframe: '30d',
  model: 'linear'
});
console.log(attribution);
```

### Track Touchpoint Example
```typescript
const response = await api.trackTouchpoint({
  creatorId: '123e4567-e89b-12d3-a456-426614174000',
  touchpointType: 'referral',
  revenue: 100.0,
  customerId: '123e4567-e89b-12d3-a456-426614174001'
});
console.log(response);
```

### Update Attribution Model Example
```typescript
const updateResponse = await api.updateAttributionModel({
  creatorId: '123e4567-e89b-12d3-a456-426614174000',
  model: 'position-based',
  weights: {
    directSales: 0.5,
    referrals: 0.3,
    collaborations: 0.2
  },
  decayRate: 0.1
});
console.log(updateResponse);
```
